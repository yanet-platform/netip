## 1.4. Контигуальные маски и CIDR

В подавляющем большинстве реальных конфигураций маска имеет специальную структуру: сначала идут все единицы, затем все нули.

**Определение.** Маска $m$ называется **контигуальной** (contiguous), если существует $k \in \{0, 1, \ldots, n\}$ такое, что:

$$m[i] = \begin{cases} 1, & \text{если } i < k \\ 0, & \text{если } i \geq k \end{cases}$$

Число $k$ называется **длиной префикса** (prefix length). Сеть с контигуальной маской длины $k$ записывается в нотации CIDR как `a/k`.

**Пример:** `10.0.0.0/8` означает сеть с адресом `10.0.0.0` и маской из 8 единиц и 24 нулей (`255.0.0.0`).

**Пример:** `2a02:6b8::/32` означает сеть с адресом `2a02:6b8::` и маской из 32 единиц и 96 нулей (`ffff:ffff::`), т.к. в IPv6 адрес кодируется 128-битным числом.

**Проверка контигуальности.** Маска контигуальна тогда и только тогда, когда все её нулевые биты расположены в хвосте:

$$m \text{ контигуальна} \iff \text{count\_zeros}(m) = \text{trailing\_zeros}(m)$$

### Арифметика по модулю $2^n$

Прежде чем сформулировать битовый критерий, уточним одну деталь. Мы работаем с $n$-битными беззнаковыми целыми числами, где арифметические операции выполняются по модулю $2^n$: если результат сложения превышает $2^n - 1$, старшие биты отбрасываются.

В Rust это поведение `wrapping_add`. Например, для $n = 32$: $\texttt{0xFFFFFFFF} + 1 = 0$. Это - корректно определённая операция в кольце вычетов $\mathbb{Z}/2^n\mathbb{Z}$.

### Теорема 3: о битовом критерий контигуальности

> Маска $m \in \{0, 1\}^n$ контигуальна тогда и только тогда, когда
>
> $$\overline{m} \mathbin{\&} (\overline{m} + 1) = 0$$
>
> где $\overline{m}$ — побитовое отрицание, а арифметика по модулю $2^n$.

**Доказательство.**

$(\Rightarrow)$ Пусть $m$ контигуальна с длиной префикса $k$, т.е. $m = \underbrace{1\ldots1}_{k}\underbrace{0\ldots0}_{n-k}$. Тогда:

$$\overline{m} = \underbrace{0\ldots0}_{k}\underbrace{1\ldots1}_{n-k} = 2^{n-k} - 1$$

Прибавление единицы даёт:

$$\overline{m} + 1 = 2^{n-k} = \underbrace{0\ldots0}_{k-1}\,1\,\underbrace{0\ldots0}_{n-k}$$

Единственный единичный бит $(\overline{m} + 1)$ приходится на нулевой бит $\overline{m}$, поэтому $\overline{m} \mathbin{\&} (\overline{m} + 1) = 0$.

Для наглядности — пример при $n = 8$, $k = 3$:

| | $m[0]$ | $m[1]$ | $m[2]$ | $m[3]$ | $m[4]$ | $m[5]$ | $m[6]$ | $m[7]$ |
|---|---|---|---|---|---|---|---|---|
| $\overline{m}$ | 0 | 0 | 0 | 1 | 1 | 1 | 1 | 1 |
| $\overline{m}+1$ | 0 | 0 | **1** | 0 | 0 | 0 | 0 | 0 |
| $\overline{m} \mathbin{\&} (\overline{m}+1)$ | 0 | 0 | 0 | 0 | 0 | 0 | 0 | 0 |

> **Примечание.** Здесь $m[i]$ нумеруется от старшего бита (MSB), как в определении контигуальности: $m[0]$ — старший бит, $m[n-1]$ — младший.

Граничные случаи:

- $k = 0$: $\overline{m} = 2^n - 1$, $\overline{m} + 1 = 0$ (переполнение mod $2^n$), $(2^n - 1) \mathbin{\&}\, 0 = 0$. ✓
- $k = n$: $\overline{m} = 0$, $\overline{m} + 1 = 1$, $0 \mathbin{\&}\, 1 = 0$. ✓

$(\Leftarrow)$ Пусть $h = \overline{m}$, $t = \text{trailing\_ones}(h)$.

Для любого целого $h \geq 0$ и любого $s \geq 0$ (теорема о делении с остатком):

$$h = \left\lfloor \frac{h}{2^s} \right\rfloor \cdot 2^s + (h \bmod 2^s)$$

Разложим $h$:

$$h = q \cdot 2^{t+1} + (2^t - 1), \quad q = h \gg (t + 1)$$

Тогда:

$$h + 1 = q \cdot 2^{t+1} + 2^t$$

Нижние $t + 1$ бит: у $h$ это $0\underbrace{1\ldots1}_t$, у $h + 1$ это $1\underbrace{0\ldots0}_t$ — их AND равен нулю. Верхние биты одинаковы, поэтому:

$$h \mathbin{\&} (h + 1) = q \cdot 2^{t+1}$$

Если $h \mathbin{\&} (h + 1) = 0$, то $q = 0$, а значит $h = 2^t - 1 = \underbrace{0\ldots0}_{n-t}\underbrace{1\ldots1}_t$ и $m = \underbrace{1\ldots1}_{n-t}\underbrace{0\ldots0}_t$ — контигуальна с префиксом $k = n - t$. $\blacksquare$

Пример неконтигуальной маски при $n = 8$: $m = 11101001$, $h = \overline{m} = 00010110$, $t = 1$, $q = 00010110 \gg 2 = 00000101 \ne 0$:

| | $m[0]$ | $m[1]$ | $m[2]$ | $m[3]$ | $m[4]$ | $m[5]$ | $m[6]$ | $m[7]$ |
|---|---|---|---|---|---|---|---|---|
| $h$ | 0 | 0 | 0 | 1 | 0 | 1 | 1 | 0 |
| $h + 1$ | 0 | 0 | 0 | 1 | 0 | 1 | 1 | 1 |
| $h \mathbin{\&} (h+1)$ | 0 | 0 | 0 | 1 | 0 | 1 | 1 | 0 |

Здесь $q \ne 0$, поэтому $h \mathbin{\&} (h + 1) \ne 0$ — маска не контигуальна.

### 1.4.1. Реализация и машинный код

Для программиста полезно видеть, во что компилируются оба варианта.

**Вариант 1** — из определения - подсчёт битов:

```rust
pub const fn is_contiguous(mask: u32) -> bool {
    mask.count_zeros() == mask.trailing_zeros()
}
```

**Вариант 2** — из теоремы - битовая арифметика:

```rust
pub const fn is_contiguous(mask: u32) -> bool {
    let hostmask = !mask;
    hostmask & hostmask.wrapping_add(1) == 0
}
```

#### x86-64 без POPCNT (generic target)

Вариант 1 вынужден эмулировать `popcount` через цепочку сдвигов и масок (алгоритм Хэмминга), что раскрывается в ~20 инструкций:

```x86asm
is_contiguous:
    mov     eax, 32
    rep bsf eax, edi            ; tzcnt (fallback через bsf)
    not     edi
    mov     ecx, edi            ; начало software popcount
    shr     ecx, 1
    and     ecx, 0x55555555
    sub     edi, ecx
    mov     ecx, edi
    and     ecx, 0x33333333
    shr     edi, 2
    and     edi, 0x33333333
    add     edi, ecx
    mov     ecx, edi
    shr     ecx, 4
    add     ecx, edi
    and     ecx, 0x0F0F0F0F
    imul    ecx, ecx, 0x01010101 ; конец software popcount
    shr     ecx, 24
    cmp     ecx, eax
    sete    al
    ret
```

Вариант 2 компилируется в **4 инструкции** (не считая `ret`):

```x86asm
is_contiguous:
    lea     eax, [rdi - 1]     ; eax = mask - 1
    or      eax, edi           ; eax = mask | (mask - 1)
    cmp     eax, -1            ; сравнить с 0xFFFFFFFF
    sete    al                 ; результат
    ret
```

> Компилятор применил эквивалентное тождество: $\overline{m} \mathbin{\&} (\overline{m} + 1) = 0 \iff m \mathbin{|} (m - 1) = 2^n - 1$, заменив NOT+ADD+AND+TEST на `lea`+`or`+`cmp`+`sete`.

#### x86-64 с POPCNT (`-C target-cpu=native`)

На процессорах с аппаратной поддержкой POPCNT/TZCNT (Intel Nehalem+, AMD Barcelona+) вариант 1 сокращается:

```x86asm
is_contiguous:
    tzcnt   eax, edi           ; eax = trailing_zeros(mask)
    not     edi                ; edi = !mask
    popcnt  ecx, edi           ; ecx = popcount(!mask) = count_zeros(mask)
    cmp     ecx, eax           ; сравнить
    sete    al
    ret
```

Вариант 2 остаётся прежним — **4 инструкции**:

```x86asm
is_contiguous:
    lea     eax, [rdi - 1]
    or      eax, edi
    cmp     eax, -1
    sete    al
    ret
```

#### IPv6 (128 бит) — generic target

Для 128-битных масок разница ещё более драматична. Вариант 1 без аппаратного POPCNT раскрывается в **~50 инструкций** (два software-popcount для старшей и младшей половин u128, два BSF, условная логика). Вариант 2 компилируется в **8 инструкций**:

```x86asm
Ipv6Network@is_contiguous:
    mov     rax, rdi
    add     rax, -1            ; lo - 1
    mov     rcx, rsi
    adc     rcx, -1            ; hi - 1 + carry
    or      rax, rdi           ; lo | (lo - 1)
    or      rcx, rsi           ; hi | (hi - 1 + carry)
    and     rcx, rax           ; объединить половины
    cmp     rcx, -1            ; сравнить с MAX
    sete    al
    ret
```

#### Сводка

| | Вариант 1 (POPCNT) | Вариант 1 (generic) | Вариант 2 (битовый) |
|---|---|---|---|
| **IPv4 (u32)** | 5 инструкций | ~20 инструкций | **4 инструкции** |
| **IPv6 (u128)** | ~12 инструкций | ~50 инструкций | **8 инструкций** |
| **ISA зависимость** | POPCNT + TZCNT | нет | нет |

## 1.5. Неконтигуальные маски: конец интервальной картины

До сих пор мы работали с контигуальными масками. Что произойдёт, если маска неконтигуальна?

**Пример 1.** Рассмотрим сеть `192.168.0.0/255.255.0.255`:

- Маска: $m = \texttt{FF.FF.00.FF}$ — третий октет свободен, остальные фиксированы.
- Адрес: $a = \texttt{C0.A8.00.00}$ (`192.168.0.0`).
- Множество адресов: $\{192.168.x.0 : 0 \leq x \leq 255\}$.

Как целые числа, эти адреса: $192.168.0.0$, $192.168.1.0$, $192.168.2.0$, $\ldots$, $192.168.255.0$. Они расположены с шагом $256$ — это не непрерывный отрезок, а 256 точек, равномерно рассеянных в диапазоне длиной $2^{16}$.

**Пример 2.** Маска $m = \texttt{F0.F0.F0.F0}$ (шестнадцатерично). Фиксированы старшие полубайты каждого октета, свободны младшие. Множество адресов «прыгает» по числовой прямой с нерегулярными промежутками.

**Пример 3 (частичное пересечение).** Возьмём две сети:

- $A = (0000, 1010)$ — фиксированы биты 0 и 2, свободны биты 1 и 3. $S(A) = \{0000, 0001, 0100, 0101\}$.
- $B = (0000, 1100)$ — фиксированы биты 0 и 1, свободны биты 2 и 3. $S(B) = \{0000, 0001, 0010, 0011\}$.

Пересечение: $S(A) \cap S(B) = \{0000, 0001\}$, но $S(A) \not\subseteq S(B)$ и $S(B) \not\subseteq S(A)$. Для контигуальных масок частичное пересечение невозможно (будет доказано в главе 2), но для неконтигуальных — возможно.

### Где встречаются неконтигуальные маски

Неконтигуальные маски — экзотика, но не выдумка. Они возникают в нескольких реальных контекстах:

**Access Control Lists (ACL).** Cisco IOS использует «wildcard masks» в ACL, где нулевые биты означают «проверять», а единичные — «игнорировать» (обратная семантика по сравнению с нашей нотацией). Wildcard mask `0.0.255.0` означает «проверять все октеты, кроме третьего» — это в точности неконтигуальная маска из примера 1 выше. Такие правила позволяют одной строкой ACL покрыть множество адресов, которое невозможно описать одним CIDR-префиксом.

**Route filtering и policy matching.** Некоторые реализации BGP policy matching позволяют задавать неконтигуальные маски для фильтрации маршрутов. Это даёт более гибкие правила, чем простой prefix match.

**Теоретическая полнота.** Даже если в конкретной задаче все маски контигуальны, *результат операции* над контигуальными сетями может потребовать неконтигуального представления. Например, разность двух контигуальных сетей, вообще говоря, не является одной контигуальной сетью. Библиотека, поддерживающая произвольные маски, может представить результат точно, а не как аппроксимацию.

**Почему нам это важно.** Поддержка неконтигуальных масок — не просто нишевая фича. Она заставляет нас строить алгоритмы, корректные *в общем случае*, а не только для удобного частного. Как мы увидим, все теоремы этой книги — о мощности, пересечении, включении — доказаны для произвольных масок. Контигуальный случай оказывается следствием, а не отдельной веткой кода.
